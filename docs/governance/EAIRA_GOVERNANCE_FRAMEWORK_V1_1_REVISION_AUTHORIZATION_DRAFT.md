## Milestone Overview

Milestone 2 is the proposed EAIRA Governance Framework v1.1 revision milestone. This draft identifies the revision scope recommended for Governance Board review.

This document is not formal authorization and does not perform any revision.

## Objective

Define the proposed v1.1 revision scope for Governance Board review using the completed dry run, improvement backlog, and change review as the authorization basis.

## Entry Criteria

- [EAIRA Governance Dry Run Report v1](./EAIRA_GOVERNANCE_DRY_RUN_REPORT_V1.md) exists and records the completed governance dry run.
- [EAIRA Governance Improvement Backlog](./EAIRA_GOVERNANCE_IMPROVEMENT_BACKLOG.md) exists and records proposed improvements traced to dry-run findings.
- [EAIRA Governance Change Review v1](./EAIRA_GOVERNANCE_CHANGE_REVIEW_V1.md) exists and records review recommendations for all backlog items.

## Authorization Basis

This draft is based on:

- [EAIRA Governance Dry Run Report v1](./EAIRA_GOVERNANCE_DRY_RUN_REPORT_V1.md)
- [EAIRA Governance Improvement Backlog](./EAIRA_GOVERNANCE_IMPROVEMENT_BACKLOG.md)
- [EAIRA Governance Change Review v1](./EAIRA_GOVERNANCE_CHANGE_REVIEW_V1.md)

## Authorized Revision Scope

If approved by the Governance Board, the v1.1 revision scope is limited to the backlog items recommended in [EAIRA Governance Change Review v1](./EAIRA_GOVERNANCE_CHANGE_REVIEW_V1.md):

- GIB-001: Define Minimum Evidence Requirements
- GIB-002: Define Status Transition Criteria
- GIB-003: Separate Metadata Dependencies
- GIB-004: Update README Reading Order
- GIB-005: Clarify Governance Review vs Dry Run
- GIB-006: Define Candidate Decision Deferred Criteria
- GIB-007: Improve Assessment Template Guidance
- GIB-008: Reviewable Status Definitions
- GIB-009: Dry Run Report Template

This scope is proposed only and becomes authorized only after Governance Board approval.

## Deferred Items

The following reviewed item is held for later consideration:

- GIB-010: Review Notes Section

## Out of Scope

The following are excluded from this draft authorization:

- Architecture
- Runtime
- Multi-AI capability approval or evaluation
- Provider Registry
- Workflow
- Adapter
- Bootstrap
- Revision execution
- Revision Plan creation
- New backlog items not recorded in [EAIRA Governance Improvement Backlog](./EAIRA_GOVERNANCE_IMPROVEMENT_BACKLOG.md)
- Changes not reviewed in [EAIRA Governance Change Review v1](./EAIRA_GOVERNANCE_CHANGE_REVIEW_V1.md)

## Deliverables

The next step is Governance Board approval review. No revision deliverable is authorized or produced by this draft.

## Exit Criteria

This draft becomes effective only after Governance Board approval.

Until approval is recorded, this draft remains a pending authorization document and does not authorize revision execution.
