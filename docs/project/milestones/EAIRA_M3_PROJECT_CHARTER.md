# EAIRA M3 Project Charter

## Objective

Establish EAIRA M3: Project Management Foundation as an active Project Layer milestone.

This charter authorizes the first controlled Project Layer validation cycle for M3 execution artifacts.

## Inputs

- [EAIRA Standard Project Task Specification](../templates/EAIRA_STANDARD_PROJECT_TASK_SPECIFICATION.md)
- [EAIRA Documentation Index Tracking System](../specifications/EAIRA_DOCUMENTATION_INDEX_SPECIFICATION.md)
- [EAIRA Milestone Strategy](../strategy/EAIRA_MILESTONE_STRATEGY.md)
- [EAIRA M2 Milestone Closure](../EAIRA_M2_MILESTONE_CLOSURE.md)
- Existing Project Layer documentation.
- Validated project experience from M2 closeout and Project Layer specification creation.

## Scope

Create or refine Project Layer execution artifacts for M3:

- [EAIRA M3 Project Charter](./EAIRA_M3_PROJECT_CHARTER.md)
- [EAIRA M3 Project Progress Report](./EAIRA_M3_PROJECT_PROGRESS_REPORT.md)
- [Project README](../README.md)
- Validation summary for the first controlled M3 cycle.

## Rules

- Markdown only.
- Project Layer only.
- Do not modify `docs/governance/`.
- Do not modify existing master specifications.
- Use master specifications only as validation references.
- Keep this charter as an authorization and planning document.
- Do not include execution status details beyond completed prerequisites.
- Do not include implementation instructions.

## Non-Goals

- Do not define governance policy.
- Do not create or modify automation code.
- Do not create `documentation-index.yaml`.
- Do not add new master specifications.
- Do not expand beyond M3 Project Management Foundation.
- Do not mix charter authorization with progress reporting.
- Do not treat Project README as the full Project Navigation capability.

## Mission

M3 exists to establish a repeatable Project Layer foundation for project authorization, progress reporting, navigation, and validation against EAIRA master specifications.

## Scope Definition

M3 is limited to Project Layer documentation artifacts and validation of those artifacts against existing Project Layer master specifications.

Project Navigation is treated as a capability. The Project README is the first navigation artifact, not the complete capability.

## Completed Prerequisites

- EAIRA Governance Framework v1.1 was published as the current governance baseline.
- EAIRA M2 was formally closed.
- EAIRA Standard Project Task Specification was documented.
- EAIRA Documentation Index Tracking System was documented.
- EAIRA Milestone Strategy was documented.

## Planned Deliverables

- M3 Project Charter.
- M3 Project Progress Report.
- Project README as the first Project Navigation artifact.
- Validation summary for conformance to the Project Layer master specifications.

## Success Criteria

- M3 has an explicit Project Layer charter.
- M3 has a factual progress report.
- Project README provides a navigation entry point.
- Charter and progress report align with the EAIRA Standard Project Task Specification.
- Project README remains a navigation artifact and does not become a specification.
- No governance documents or master specifications are modified.

## Acceptance Criteria

- Required charter sections are present.
- Completed prerequisites are separate from planned deliverables.
- Scope is Project Layer only.
- Project Navigation is identified as a capability.
- Project README is identified as an artifact.
- No automation is implemented.
- No documentation index output is created.

## Final Decision

Decision: DOCUMENTED

Summary:
M3 Project Charter is documented as the authorization and planning artifact for EAIRA M3: Project Management Foundation.
