# EAIRA Phase Closeout Template

## Objective

Record the formal closure of an EAIRA project phase.

## Inputs

- Phase charter or authorization artifact.
- Progress report or execution history artifact.
- Validation evidence.
- Related project navigation or strategy artifacts.

## Scope

Use this template for Project Layer phase closeout records.

Phase Closeout is a lifecycle artifact, not a milestone artifact.

## Rules

- Markdown only.
- Project Layer only.
- Record completed and verifiable facts.
- Reference supporting artifacts instead of duplicating them.
- Keep the closeout record immutable after creation except for correction tasks.
- Limit Lessons Learned to at most three lessons.

## Non-Goals

- Do not define governance policy.
- Do not create execution instructions.
- Do not revise strategy, specifications, validation evidence, or progress history.
- Do not use closeout records as current-state reports.

## Phase Summary

State the phase being closed, its purpose, and the basis for closure.

## Completed Deliverables

List completed deliverables and link to the owning artifacts where possible.

## Acceptance Review

Record whether the phase met its acceptance criteria.

## Phase Exit Criteria

Explain why the phase can be exited.

## Evidence Summary

Summarize the evidence chain supporting closure. Reference evidence artifacts rather than reproducing them.

## Transition Items

List the items handed to the next phase.

## Lessons Learned

Use at most three lessons.

Each lesson must use:

- Observation
- Evidence
- Impact
