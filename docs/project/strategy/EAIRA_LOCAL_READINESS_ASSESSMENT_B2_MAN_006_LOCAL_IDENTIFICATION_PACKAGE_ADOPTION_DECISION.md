## 7. Adopted Fail-Closed Boundaries

A future Stage 2 session must stop if:

- baseline branch or commit differs;
- the working tree is not clean;
- the active path is under OneDrive;
- no suitable WSL distribution is selected;
- WSL mapping differs from the documentary target;
- Git resolves as an alias, function, wrapper, shim, launcher or ambiguous
  candidate;
- executable provenance is unresolved;
- option applicability is unsupported;
- relevant environment overrides are unexpectedly set;
- repository discovery differs from the selected root;
- configuration output exceeds the specifically authorized named keys;
- hook or fsmonitor behavior cannot be bounded;
- an inspection operation may start or mutate a daemon;
- evidence-root ACL is broader than the selected template;
- BitLocker protection cannot be verified;
- an operation requests unapproved elevation;
- network activity occurs;
- stderr is non-empty unless expressly expected;
- output contains credentials, secrets, PII, file contents or other prohibited
  material;
- repository, configuration, ACL, encryption, process, service, hook, socket
  or filesystem state changes;
- operator/verifier separation is not established; or
- independent verification cannot be completed.

A stopped session grants no retry and satisfies no acceptance category.
